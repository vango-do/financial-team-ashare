# 贡献指南

感谢你为「金融团队（A股版）」做贡献。为了让协作稳定可复现，请在提交前完成以下流程。

## 提交流程

1. Fork 本仓库并创建分支，分支命名建议：`feat/*`、`fix/*`、`docs/*`。
2. 在本地执行依赖安装与基础检查：
   ```bash
   poetry install
   poetry run python -m pytest
   ```
3. 确认新增功能或修复包含必要说明（README 或注释）。
4. 提交信息尽量清晰，推荐格式：`type(scope): summary`。
5. 发起 Pull Request，说明变更内容、影响范围、验证方式。

## 代码与文档规范

- 默认使用 UTF-8 编码。
- Python 代码保持清晰、可读、可复现，避免硬编码私有路径。
- A 股规则相关逻辑（T+1、涨跌停）修改时，请补充测试或最小复现说明。
- 涉及 Prompt 或决策逻辑修改时，请说明对“风格隔离”和“证据优先”的影响。

## 安全要求（强制）

请在提交前逐项检查：

1. 不提交 `.env`、API Key、Token、Cookie、私有网关地址。
2. 不提交桌面导出报告、运行日志、缓存数据。
3. 使用 `git status` 与 `git diff` 二次确认变更文件。
4. 如发现密钥泄露，请先旋转密钥再提交修复。

## Pull Request 建议模板

请在 PR 描述中包含：

- 变更目的
- 变更文件清单
- 本地验证步骤
- 风险与回滚方案（如有）
