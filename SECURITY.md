# 安全策略

## 支持范围

本项目当前主分支（`main`）为安全修复优先支持分支。历史分支不承诺持续安全更新。

## 报告安全问题

如发现漏洞、密钥泄露或数据安全风险，请不要在公开 Issue 中直接披露细节。

- 邮箱：`juzhouq@gmail.com`
- 标题建议：`[Security] 简要问题描述`
- 建议附带：影响范围、复现步骤、风险等级、修复建议

我们会在收到报告后尽快确认并安排修复。

## 敏感信息处理规范

- 禁止提交 `.env`、API Key、Token、Cookie、私有网关地址。
- 若发现密钥泄露，请立即旋转密钥并清理历史提交。
- 涉及第三方接口调用时，优先使用环境变量注入配置。

## 依赖与供应链建议

- 合并前执行依赖升级与漏洞扫描。
- 对高风险依赖进行版本锁定与变更审查。
- 生产环境尽量采用最小权限原则管理 API Key。
