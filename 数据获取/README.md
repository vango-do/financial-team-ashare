# 东方财富 A 股模板自动填充工具

## 1. 项目是怎么完成的

这个项目基于你已经整理好的模板文件完成了三件事：

1. 读取 `专家分析数据模板.xlsx`，按每一行的 `字段英文名` 定位需要填充的数据。
2. 使用 `SCRAPLING_CN_LIB`（或仓库内 `./scrapling_cn_lib`）提供的 `scrapling` 抓取能力，从东方财富多个接口动态获取数据。
3. 将抓取结果写入每个 sheet 的 `示例值(留空)` 列，并导出为“股票名称命名”的新 Excel。

主脚本文件：

- `eastmoney_excel_filler.py`

## 2. 程序功能

运行后会：

1. 询问 6 位 A 股代码（例如 `603305`）。
2. 自动判断市场（SH/SZ/BJ）。
3. 抓取并组装数据：
   - 行情（最新价、开高低收、成交量、PE/PB、市值等）
   - 财务三表核心字段（收入、利润、资产负债、现金流）
   - 资金流向（主力/大单/中单/小单相关快照）
   - 公告（最新公告标题、时间、来源）
   - 股东/高管增减持
   - 北向相关（20 日净流入、持仓变化）
4. 计算补充字段（如 FCF、ROIC、PEG、内在价值估计等）。
5. 写回所有 sheet 的 `示例值(留空)` 列。
6. 输出新文件，文件名为股票名称（重名自动加 `_1`、`_2` 后缀）。

## 3. 速度优化（本次重点）

为了提速，脚本做了这些优化：

1. **并发抓取**：把互相独立的数据源改为并发请求（`ThreadPoolExecutor`）。
2. **请求缓存**：同一轮运行内相同 URL+参数命中本地缓存，避免重复请求。
3. **一次性字段拉取**：尽量在单个接口请求中拿到更多字段，减少请求数。

## 4. 反爬与稳定性策略

在提速同时，加入了“温和防封”机制：

1. **温和并发**：并发数限制为 4，避免瞬时高压请求。
2. **抖动延时**：每次请求前加入短随机抖动，降低固定节奏特征。
3. **指数退避重试**：失败后自动退避重试，不盲目快速重打。
4. **浏览器指纹轮换**：`impersonate` 在 `chrome/edge` 间轮换。
5. **反爬识别**：识别 403/429/503 及常见拦截文案，触发重试逻辑。

## 5. 使用方法

在当前目录执行：

```powershell
python .\eastmoney_excel_filler.py
```

然后输入股票代码即可。

## 6. 输入与输出

输入文件（模板）：

- `专家分析数据模板.xlsx`

输出文件示例：

- `旭升集团.xlsx`
- `旭升集团_1.xlsx`（当同名文件已存在时）
